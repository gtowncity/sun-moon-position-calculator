# ALGORITHMS.md

## Azimuth

Azimuth is expressed in degrees clockwise from geographic true north:

- 0 degrees = North
- 90 degrees = East
- 180 degrees = South
- 270 degrees = West

Magnetic north is not used.

## Altitude / Elevation

Altitude is the angular height above the ideal mathematical horizon. Positive values are above the horizon; negative values are below it.

The result model separates:

- geometric altitude: no atmospheric refraction;
- apparent altitude: geometric altitude plus the selected refraction treatment.

## Zenith Angle

Zenith angle is calculated as:

```text
90 degrees - altitude
```

The app reports both geometric and apparent zenith angles.

## Topocentric and Geocentric

Topocentric coordinates are calculated for the concrete observer on Earth, including latitude, longitude and elevation. This is required for the Moon because lunar parallax is large enough to be visible in normal user-facing results.

Geocentric coordinates are relative to Earth's center and are not sufficient for the Moon position shown in the UI.

## Apparent and Geometric

Geometric values exclude atmospheric refraction. Apparent values include the selected refraction model. Other apparent-coordinate corrections depend on the underlying ephemeris/runtime library.

## NREL SPA

NREL SPA is documented as the target solar reference method. The current runtime does not redistribute NREL source code and does not claim to be a complete SPA port. See `ACCURACY.md`.

## Astronomy Engine

`astronomy-engine` provides the runtime ephemeris functions used for topocentric equatorial and horizontal coordinates, lunar illumination, lunar phase and rise/set/hour-angle searches.

## Time and Time Scales

User input is local civil time in an IANA time zone. The app uses `@js-temporal/polyfill` to convert local date/time to `Temporal.Instant`.

DST behavior:

- Non-existing local times during spring-forward are rejected.
- Ambiguous local times during fall-back are reported and resolved to the earlier instant.
- Intervals are generated by adding durations to UTC instants, not by naive local wall-clock stepping.

## Refraction

Modes:

- none: apparent equals geometric;
- standard: `astronomy-engine` normal refraction;
- custom: Bennett-style correction scaled by pressure and temperature.

Default custom values are 1013.25 hPa and 15 degrees Celsius.

## Rise, Set and Transit

Rise and set are searched per body for the selected local date window. The event result is marked `not_found` if no event is found inside the local date.

Transit is searched as hour angle 0 and filtered to the selected local date window.

Polar regions and lunar dates without rise/set are valid outcomes, not calculation crashes.

## Effective Imaging Time

The Astro night planner classifies already computed topocentric solar positions by geometric Sun altitude:

- Sun above -6 deg: not useful for faint deep-sky imaging.
- Sun <= -6 deg and > -12 deg: limited twilight use.
- Sun <= -12 deg and > -15 deg: usable for bright targets or tests.
- Sun <= -15 deg and > -18 deg: good, but not full astronomical night.
- Sun <= -18 deg: real astronomical night.

Selectable imaging modes:

- strict DSO: Sun <= -18 deg
- balanced DSO: Sun <= -15 deg
- bright target/test: Sun <= -12 deg

The night planner uses interval samples generated from real `Temporal.Instant` values. Twilight boundary times are linearly interpolated between adjacent computed Sun-altitude samples. This is a UI planning summary; smaller intervals produce more precise boundary estimates.
